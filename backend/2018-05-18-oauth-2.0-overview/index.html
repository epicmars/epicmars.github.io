OAuth 2.0综述

OAuth 2.0是用于授权的工业标准协议。OAuth 2.0取代了创建于2006年的OAuth协议。OAuth 2.0在专注于客户端开发者的精简性的同时也为web应用、桌面应用、移动手机和家用设备提供了专用的授权流。

OAuth 2.0授权框架-RFC 6749

OAuth 2.0授予类型（Grant Types）

OAuth 2.0框架为不同的使用情况制定了若干授予类型，以及一个用于创建新授予类型的框架。最常用的OAuth 2.0授予类型如下：

  • 授权码（Authorization Code）

    授权码授予类型被机密和公开客户端用于使用授权码交换一个访问令牌（access token）。

    在用户通过重定向URL返回客户端后，应用会从该URL获取授权码，并使用它请求一个访问令牌。

  • [隐式授予（Implicit Grant）]

    隐式授予是一个可用于公开客户端的简化授权流，其中不需要额外的授权码交换步骤，而是立即返回一个访问令牌。

    通常是不建议使用隐式授予流的（有些服务器完全禁止这种流）。在该规范最初编写的时候，工业界的最佳实践已经转变到建议公开客户端应当这样做，要么在没有客户端secret的时候使用授权码，或者使用PKCE扩展

  • 密码

OpenID

OpenID Connect是构建于OAuth 2.0协议上的一个简单身份层。基于一个授权服务器提供的认证服务，它可以让客户端识别终端用户的身份，以及使用一种互操作和类REST的方式获取终端用户的基本资料信息。
