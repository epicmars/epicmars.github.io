关键字和保留字

ECMA-262描述了一组具有特定用途的关键字，这些关键字可用于表示控制语句的开始或结束，或者用于执行特定操作等。按照规则，关键字也是语言保留的，不能用作标识符。以下就是ECMAScript的全部关键字：

break	do	instanceof	typeof
case	else	new	var
catch	finally	return	void
continue	for	switch	while
debugger	function	this	with
default	if	throw	
delete	in	try

ECMA-262还有另一组不能用作标识符的保留字。

变量

ECMAScript的变量是松散类型的，所谓松散类型就是可以用来保存任何类型的数据。换言之，每个变量仅仅是一个用于保存值的占位符而已。定义变量使用var操作符。如下所示：

var message;

使用var关键字定义的变量将成为定义该变量的作用域中的局部变量。也就是说，如果函数中使用var定义了一个变量，那么这个变量在函数退出后就会被销毁。

数据类型

ECMAScript中有5基本数据类型：

Undefined
Null
Boolean
Number

以及一种复杂数据类型：

Object

Object本质上是由一组无序的名称-值对组成。ECMAScript不支持任何创建自定义类型的机制，而所有值最终都是上述6种数据类型之一。

typeof操作符

鉴于ECMAScript是松散类型的，因此需要有一种手段来检测给定变量的数据类型。typeof就是负责提供这方面的信息的操作符。

操作符

一元操作符

布尔操作符

布尔操作符!的返回值总是一个布尔值。 布尔操作符&&和||的返回值不一定是Boolean类型，其返回值总是操作数表达式中的某一个。

逻辑非（!）

可以应用于ECMAScript中的任何值，无论这个值是什么数据类型，这个操作符都会返回一个布尔值。 当操作数符合以下情况时逻辑非操作符返回true: 空字符串 0 null NaN undefined 当操作数符合以下情况时逻辑非操作符返回false: 一个对象 非空字符串 非0数值

逻辑与（&&）

如果有一个操作符不是布尔值，求值为false的返回false，否则返回最后使表达式求值为true的操作符。两个操作符中有一个为null,NaN,undefined时分别返回null,NaN,undefine。

逻辑或（||）

与逻辑与类型，如果有一个操作数不是布尔值，逻辑或也不一定返回布尔值：根据短路操作符原理，返回首先使表达式求值为true得操作数，两个操作数中两个都为null,NaN,undefined时分别返回null,NaN,undefine。

利用逻辑或的短路行为来避免为变量赋null或undefined值：

var myObject = preferredObject || backupObject;

myObject被赋值给两个变量中的一个，变量backupObject。

乘性操作符

3个乘性操作符：乘法、除法和求模。这些操作符与Java、C或Perl中的相应操作符用途类似，不过在操作数为非数值时会使用Number()函数执行自动类型转换。

加性操作符

加法

与乘性操作符类型，加法操作符也会在后台转换不同的数据类型。如果有一个操作数是字符串，那么按照字符串拼接的方式处理，其中如果另一个操作数是对象、数值或布尔值则调用它们的toString()方法，对于undefined和null，则调用String()函数并取得字符串"undefined"和"null"。

减法

如果有一个操作数是字符串、布尔值、null或undefined，则先用Number()函数将其转换为数值。如果有一个操作数为对象，则调用对象的valueOf()方法取得该对象的数值，如果对象没有valueOf()方法，则调用toString()方法将得到的字符串转换为数值。

关系操作符

大于（>）、小于（<）、小于等于（<=）、大于等于（>=）

相等操作符

相等和不相等：进行类型转换 全等和不全等：不进行类型转换

语句

switch语句

switch语句可以使用任何数据类型，每个case语句的值不一定是常量，可以是常量，甚至表达式。在比较值时使用的是全等操作符，因此不会发生类型转换。

函数

ECMAScript中的函数使用function关键字来声明，定义时不必指定是否返回值，实际上，任何函数在任何时候都可以通过return语句后跟要返回的值来实现返回值。

理解参数

ECMAScript中所有的参数传递都是值，不可能通过引用传递参数。

没有重载

函数不存在签名，其参数是由包含零个和多个值的数组来表示的，真正的重载不可能做到。如果在ECMAScript中定义了两个同名的函数，则该名字只属于后定义的函数。

一种解决方法是通过检查传入参数的类型和数量并作出不同的反应来模仿方法的重载。
