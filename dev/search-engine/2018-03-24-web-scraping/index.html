网络爬虫概述

爬取之前

  • 了解相关法律，爬取合乎法律的内容
  • 对要爬取的站点进行适当背景调查，如检查robots.txt文件，网站地图，估算网站大小，识别网站所用的技术等

开始爬取

  • 选择一些要爬取站点的起始地址
  • 从一个地址开始，下载页面
    • 设置Http请求头中的“User-Agent”字段
    • 下载错误时重试下载
    • 如果爬取的站点从当前网络不可达，可以支持网络代理连接
    • 设置爬取频率，即两次页面下载之间应当设置适当的延时，避免对爬取的站点造成太大压低，否则有被封禁的风险
    • 避免陷入爬虫陷阱，一些动态页面可能会递归地生成出无限的页面，可以适当限制爬取的页面的深度，即到达当前页面时已经访问了多少个链接
    • 对于python爬虫，如果追求速度，不建议使用Http请求库requests，速度比自带的urllib2慢了好几倍

数据抓取

  • 分析希望获取的数据
  • 在页面下载完成后，可以采用三种方式提取需要的数据：
    1. 正则表达式
    2. BeautifulSoup，可以对下载的HTML文档进行格式校准，并根据HTML标签等定位内容
    3. lxml，也可以解析格式不完整的HTML文档，并可以采用css选择器或xpath选择器提取数据

其中正则表达式性能最快，但在页面内容变化时容易匹配失效，如某一标签添加了某一属性等。 lxml性能也很快，并且与BeautifulSoup一样直接解析HTML文档，对于页面内容的变化影响小，除非页面结构也发生了较大的变化。

缓存

  • 采用数据库缓存，如MySQL，MongoDB等，一般的网页爬取采用非关系型的NoSQL数据库更合适，这里采用MongoDB，同时为了避免重复插入同一网页以及加快查询效率，需要对页面url字段创建唯一性约束的索引或者直接将url存到_id字段。

  • 页面内容可能发生了变化，那么当前缓存可能已经过期，设置一个缓存失效时间，在缓存过期后重新下载页面。MongoDB中可以创建一个具有一个存活期TTL（time to live）的索引，在TTL超时后会自动删除该文档。

  • 数据压缩，像taobao这样的一个网站就有上亿的页面，当爬取大量页面时，为了节约磁盘空间，在缓存前可以将下载的页面进行压缩后再保存，python中可以使用zlib库。

动态内容

相对于使用浏览器所展示的页面内容，使用爬虫下载同一链接下的HTML源码，两者的内容可能不一致，因为大多数网站都在前端使用脚本语言（如JavaScript）进行内容的动态加载，即AJAX技术，而直接下载的HTML源码是静态的内容。有两种方式可以用于在爬虫中进行这种动态加载以获取完整的页面内容：

  • JavaScript逆向工程 直接通过动态加载脚本获取数据，并且解析返回的数据即可。
  • 渲染JavaScript 对于非常复杂的网站，逆向工程也会较为复杂，另外一种较为直接的方式是使用浏览器渲染引擎来避免这一工作，渲染引擎是浏览器在显示网页时解析HTML、应用CSS样式并执行JavaScript语句的部分。
    • PyQt和PySide，使用Qt框架提供的Web容器来加载页面，然后获取HTML。
    • Selenium，PyQt的使用较为灵活但也更复杂，可以将页面的加载交给本地的浏览器，Selenium就提供了使浏览器自动化的API接口，即可以自动加载需要的页面并与之进行交互。
    • PhantomJs，上面两种都需要一个界面来展示加载的页面，而PhantomJs正好解决了这一点，它是一个可编程的WebKit浏览器引擎，不需要界面也能运行，配合Selenium简直完美。PhantomJs负责渲染页面，而Selenium负责交互，不需要任何界面。

高并发与分布式爬虫

根据最近的爬取的站点来看，即使是个人博客站点，页面数从上百到上万都有。相比各种本地IO，受网络带宽和服务器带宽等限制，网络IO属于速度最慢的一种，如果采用单线程的方式，假设以常规100ms一个页面的速度爬取100万个页面，需要大约28个小时，视网络状况和页面内容大小，实际上通常比这个更慢。如果采用多线程的方式，例如开启100个线程去进行爬取，那么只要网络带宽足够，内存足够，CPU性能够用，理论上速度可以提高100倍，半个小时可以爬完。

使用100个线程并发，每秒会发起1000个请求，对于一般站点都会造成较大带宽、内存、CPU资源的占用，会有被封禁的风险。根据爬取站点的特点可以进行如下策略分配：

  1. 但如果仅仅是爬取单个站点的内容，可以考虑使用分布式爬虫和ip池技术，这个后续再谈
  2. 如果爬取多个站点，可以考虑对不同站点从时间上进行交叉爬取，避免将压力同时施加到单一站点

关于增加爬取效率，还可以考虑使用IO复用技术，可以降低内存占用。
