React Native之Android开发：原生UI组件

参考Native UI Components

原生APP中有成千上万的UI小部件可供使用，这其中包括平台自身的、第三方的以及自定义。React Native已经包装了若干最关键的平台组件，例如ScrollView和TextInput，但这并没有包括所有UI组件，也没有你自己定义的组件。幸运的是，可以容易将这些已有组件无缝地集成到React Native应用中。

假设你已经对Android开发较为熟悉，本文将会向你示范如何创建一个原生UI组件，带领你过一遍React Native库中已有ImageView组件的一个子集的实现。

ImageView示例

为了允许在JavaScript中使用ImageView，我们将过一遍所需要的实现。

通过继承ViewManager或更常用的SimpleViewManager来创建和操纵原生View。SimpleViewManager较为方便，是因为它应用了常用属性，例如background color, opacity, 和Flexbox layout。

这些子类本质上都是单例，它们将原生view发布到NativeViewHierarchyManager，该类会代理回它们以根据需要设置和更新属性。ViewManagers通常是view的代理，通过桥将事件发回给JavaScript。

发布一个视图很简单：

  1. 创建ViewManager的子类
  2. 实现createViewInstance方法
  3. 使用@ReactProp（或@ReactPropGroup）注解暴露view的属性setter方法
  4. 在applications包的createViewManagers方法中注册管理器
  5. 实现JavaScript模块

1. 创建ViewManager子类

public class ReactImageManager extends SimpleViewManager<ReactImageView> {

  public static final String REACT_CLASS = "RCTImageView";

  @Override
  public String getName() {
    return REACT_CLASS;
  }

2. 实现createViewInstance方法

  @Override
  public ReactImageView createViewInstance(ThemedReactContext context) {
    return new ReactImageView(context, Fresco.newDraweeControllerBuilder(), mCallerContext);
  }

3. 使用@ReactProp（或@ReactPropGroup）注解暴露view的属性setter方法

Setter方法将要更新的view（当前视图类型的）作为第一个参数并且属性值为第二个参数。Setter方法应当定义为void方法，并且应当是public的。发送给JS的属性类型根据setter的参数值自动进行确定。目前支持如下类型的值：boolean, int, float, double, String, Boolean, Integer, ReadableArray, ReadableMap。

重点! 在ReactJS中，更新属性值会导致setter方法调用。注意，我们更新组件的一种方式就是移除已经设置过的属性，在这种情况下，setter方法会被调用，并且会通知view manager属性已经更改。这种情况下会提供“默认”值（对于基本类型，“默认”值可以通过defaultBoolean, defaultFloat指定，例如，@ReactProp注解的参数，对于复杂类型，setter方法会被调用并且值设置为null）。

  @ReactProp(name = "src")
  public void setSrc(ReactImageView view, @Nullable ReadableArray sources) {
    view.setSource(sources);
  }

  @ReactProp(name = "borderRadius", defaultFloat = 0f)
  public void setBorderRadius(ReactImageView view, float borderRadius) {
    view.setBorderRadius(borderRadius);
  }

  @ReactProp(name = ViewProps.RESIZE_MODE)
  public void setResizeMode(ReactImageView view, @Nullable String resizeMode) {
    view.setScaleType(ImageResizeMode.toScaleType(resizeMode));
  }

4. 注册ViewManager

Java中的最后一步是注册ViewManager到应用中，这和原生模块中的注册方式类似，通过application package的成员方法createViewManagers来注册。

  @Override
  public List<ViewManager> createViewManagers(
                            ReactApplicationContext reactContext) {
    return Arrays.<ViewManager>asList(
      new ReactImageManager()
    );
  }

5. 实现JavaScript模块

最后的最后是创建JavaScript模块，它为你的新的view的用户定义了Java和JavaScript间的交互层。大多数的工作都由Java和JavaScript中的React代码完成了，剩下的工作就交给你去描述propTypes。

// ImageView.js

import PropTypes from 'prop-types';
import {requireNativeComponent, ViewPropTypes} from 'react-native';

var iface = {
  name: 'ImageView',
  propTypes: {
    src: PropTypes.string,
    borderRadius: PropTypes.number,
    resizeMode: PropTypes.oneOf(['cover', 'contain', 'stretch']),
    ...ViewPropTypes, // include the default view properties
  },
};

module.exports = requireNativeComponent('RCTImageView', iface);

requireNativeComponent通常接收两个参数，第一个是原生view的名称，第二个是描述组件接口的对象。组件接口应当定义一个友好的name，以便于输出调试消息，并且必须定义由原生View反射的propTypes。propTypes用于检查用户对原生view的使用的有效性。注意，如果你需要你的JavaScript组件不仅仅指定name和propTypes，还可以进行类似自定义事件的处理，你可以将原生组件包装到一个常规的react组件中。在这种情况下，你会在requireNativeComponent中传入包装组件而不是iface。在下面的MyCustomView示例中可以看到。

事件(Events)

现在我们知道了如何暴露原生视图组件，使得我们可以容易地在JS中进行控制。 但是我们如何处理来自用户的时间，例如pinch-zooms或panning。当一个原生事件发生了，原生代码会发出一个事件给代表视图的JavaScript，这两个view通过getId()方法的返回值进行链接。

class MyCustomView extends View {
   ...
   public void onReceiveNativeEvent() {
      WritableMap event = Arguments.createMap();
      event.putString("message", "MyMessage");
      ReactContext reactContext = (ReactContext)getContext();
      reactContext.getJSModule(RCTEventEmitter.class).receiveEvent(
          getId(),
          "topChange",
          event);
    }
}

为了将topChange事件的名称映射到JavaScript中的onChange回调属性，通过重写ViewManager中的getExportedCustomBubblingEventTypeConstants方法来注册它。

public class ReactImageManager extends SimpleViewManager<MyCustomView> {
    ...
    public Map getExportedCustomBubblingEventTypeConstants() {
        return MapBuilder.builder()
            .put(
                "topChange",
                MapBuilder.of(
                    "phasedRegistrationNames",
                    MapBuilder.of("bubbled", "onChange")))
                    .build();
    }
}

该回调使用原始事件进行调用，为了得到一个更简单的API，我们通常在包装组件中进行处理：

// MyCustomView.js

class MyCustomView extends React.Component {
  constructor(props) {
    super(props);
    this._onChange = this._onChange.bind(this);
  }
  _onChange(event: Event) {
    if (!this.props.onChangeMessage) {
      return;
    }
    this.props.onChangeMessage(event.nativeEvent.message);
  }
  render() {
    return <RCTMyCustomView {...this.props} onChange={this._onChange} />;
  }
}
MyCustomView.propTypes = {
  /**
   * Callback that is called continuously when the user is dragging the map.
   */
  onChangeMessage: PropTypes.func,
  ...
};

var RCTMyCustomView = requireNativeComponent(`RCTMyCustomView`, MyCustomView, {
  nativeOnly: {onChange: true}
});
