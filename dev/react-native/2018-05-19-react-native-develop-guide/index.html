React Native开发基础知识

目前React Native的最新版本为0.55。

开发模式环境

完全使用JavaScript进行开发

如果不需要自定义原生代码，完全使用JavaScript进行开发，那么可以采用官方文档的Quick Start中介绍的方法，使用Create React Native App创建工程后，在开发阶段，启动应用服务，通过在手机上安装Expo客户端，就可以通过从线上加载的方式运行所开发的应用了。在发布阶段，则通过一般打包发布流程进行安装包的分发。

Expo是各种工具、库和服务的集合，Expo SDK是一个原生及JS库，可以提供对不同移动操作系统功能的访问，例如相机、通讯录、本地存储以及其它硬件，这是因为它本身包含了原生代码库完成这些工作，通过集成Expo SDK，那么就不再需要自定义的原生代码来完成这一工作，纯粹使用JavaScript就可以编写原生App了。对于简单的应用这完全够用了，并且可以免去适配不同操作系统的工作，但Expo SDK只能兼顾部分最常用的功能，因此对于复杂或特定领域的应用可能仍然需要采用混合式开发的方式。

使用JavaScript和原生代码进行混合式开发

首先需要安装依赖环境和工具，以下是Mac中所使用的命令行：

  • 安装NodeJs，一个基于 Chrome V8 引擎的 JavaScript 运行时。

      brew install node

    或者下载安装包进行安装。

  • 安装Watchman，Facebook用于观察文件系统变化的工具。

      brew install watchman

  • 安装React Native CLI

      npm install -g react-native-cli

  • 安装JDK，目前Android Studio安装后自带OpenJDK，在其安装目录中可以找到

  • 安装Android开发环境，如Android Studio和Android SDK

    构建React Native应用需要指定使用Android 6.0(Marshmallow) Android SDK Build-Tools应当指定为23.0.1

集成到已有App中

除了新建一个React Native工程，还可以将其集成到已有原生工程中。具体步骤参考Integration with Existing Apps。

基础概念

React Native和React十分相似，但它使用原生组件而非Web组件作为构建模块。因此需要理解基本的React概念，例如JSX，组件，state，props。如果你已经熟悉React，那么你仍然需要学习一些特定于React-Native的内容。

import React, { Component } from 'react';
import { Text } from 'react-native';

export default class HelloWorldApp extends Component {
  render() {
    return (
      <Text>Hello world!</Text>
    );
  }
}

JSX

一种在JavaScript中嵌入XML的语法。许多框架使用一个特殊的模板语言来在标记语言中嵌入代码。在React中这一点是相反的，JSX允许你在代码中嵌入标记语言。

Component（组件）

上面的HelloWorld代码中定义了一个HelloWorldApp，它是一个新的Component，在构建一个React Native应用时常常会创建新的组件。任何屏幕上可以看到的东西某种程度上都是一个组件。一个组件可以很简单，唯一需要的是一个render函数，它返回一些JSX用于渲染。

Props（属性）

大多数组件在创建时都可以使用不同的参数进行自定义。这些创建参数称为props，即组件的属性。组件也可以使用这些props,这可以使得单个组件可以在app的不同地方进行复用。

State（状态）

控制一个组件的数据有两种：props和state。props由父组件设置并且在一个组件的生命周期中保持不变，对于变化的数据必须使用state。

通常应当在构造器中初始化state，然后在改变它时调用setState方法。

样式与布局

Style（样式）

使用React Native不需要一种特定的语言或语法来定义样式。使用JavaScript进行样式的定义就可以了。所有核心组件都接收一个名为style的属性名。样式名称和值通常和Web页面中CSS中的类似，只是名称使用驼峰命名法，如背景颜色backgroundColor而不是background-color。

高度（Height）和宽度（Width）

组件的高度和宽度确定了其在屏幕上的尺寸。

  • 固定尺寸(Fix Dimension)

    React Native中所有尺寸都是无单位的，代表密度无关像素。

  • 弹性尺寸(Flex Dimension)

    在组件的style中使用flex可以让组件根据可用空间进行动态地伸缩。通常你会使用flex: 1，它告诉一个组件去填满所有可用空间，给出的flex值越大，那么组件占有可用空间的比例也越大。

    一个组件只能在父布局的尺寸大于0时才可以进行扩展并填充所有可用空间。如果一个父布局即没有固定尺寸，也没有弹性尺寸，那么该父布局的尺寸为0，因此其flex子布局是不可见的。

Flexbox布局

一个组件可以使用flexbox算法指定其子视图的布局。Flexbox是设计用于在不同尺寸屏幕上提供一致性布局。

通常会使用flexDirection，alignItems，justifyContent来达到正确的布局。

  • Flex方向（Flex Direction） 指定布局的主坐标轴（primary axis）
  • 内容对齐（Justify Content） 指定子视图沿主坐标轴的分布（distribution）。子视图应当分布在开头、中间、尾部、或者空间均匀。
  • 排列条目（Align Items） 指定子视图沿从坐标轴（secondary axis，如果主坐标轴是column，那么从坐标轴就是row）的对齐方式。

更进一步，除了上面的基本弹性布局方式，还有许多用于布局的样式，控制布局的属性的全部文档见此处。

常用组件

文本输入（Handling Text Input）

TextInput是允许用户进行输入的基本组件。

触控处理（Handling Touches）

移动App的用户交互主要通过触控完成。用户可以使用各种手势组合，例如点击按钮，滚动一个列表或者缩放地图。React Native提供了处理所有常见类型手势的组件，以及一个综合性的手势响应系统（gesture responder system）用于更高级的手势识别。

滚动视图（ScrollView）

ScrollView是一个通用的滚动容器，可以容纳多个组件和视图。滚动项不一定是同类型，并且可以在水平和垂直方向进行滚动（通过设置horizontal属性）。

ScrollView主要用于展示少量具有有限条目的内容。即使部分不可见，ScrollView的所有元素和子视图都会被渲染。因此，如果需要展示长的列表，应该使用列表视图。

列表视图（List View）

React Native提供了一套组件用于展示数据列表。通常你会使用FlatList或者SectionList。

与ScrollView不同，FlatList只会渲染屏幕上显示的元素，而非全部元素。

网络

许多app都需要从远程URL加载资源。你可能会向一个REST API发起一个POST请求，或者你可能简单地从另一个服务器获取一块静态内容。

  • 使用Fetch

    React Native提供了Fetch API用于网络编程。Fetch与XMLHttpRequest或其它网络API有些相识。你可以参考MDN上的Fetch文档。

  • 使用其它网络库

    XMLHttpRequest API是内置于React Native中的，这意味着你可以使用依赖于它的第三方库，例如frisbee或者axios。

  • WebSocket支持

    React Native也支持WebSockets，一个在单个TCP连接上提供全双工通信信道的协议。
