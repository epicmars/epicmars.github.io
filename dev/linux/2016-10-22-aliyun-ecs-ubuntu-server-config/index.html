阿里云ECS之Ubuntu服务器配置

连接到服务器

购买阿里云ECS时设置的管理员密码就是root密码了，可以用于ssh远程登录，使用秘钥登录的方式肯定更方便，那么首先登录服务器，看看有没有什么要准备的。

首先使用root登录，生成的主机名是一串编码，换成自己的主机名：

vi /etc/hostname

用户于组管理

然后添加自己的用户组，可以使用较为高级的useradd，groupadd，usermod工具，也可以使用更初级的adduser和addgroup工具。

addgroup newgroup
# 添加newuser并指定其主用户组为newgroup
adduser newuser --ingroup newgroup
# 加入到超级用户组
adduser newuser sudo
# 设置用户密码
passwd newuser

安装Zsh与oh-my-zsh

首先需要安装zsh, 由于on-my-zsh使用git仓库进行更新还需要安装git:

sudo apt-get install zsh git

  • 问题1 sudo: unable to resolve host your-hostname

    修改/etc/hosts，添加一行： 127.0.1.1 your-hostname

  • 问题2 Reading package lists… Done Building dependency tree Reading state information… Done E: Unable to locate package zsh

    更新源： sudo apt-get update

  • 问题3 perl: warning: Setting locale failed. perl: warning: Please check that your locale settings: LANGUAGE = (unset), LC_ALL = (unset), LC_CTYPE = “zh_CN.UTF-8”, LANG = “en_US.UTF-8” are supported and installed on your system. perl: warning: Falling back to the standard locale (“C”). 解决方法

使用新用户重新登录服务器，终端用起也不够爽，果断换oh-my-zsh。 oh-my-zsh

sh -c "$(wget https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh -O -)"

远程登录

ubuntu默认没安装openssh-server，设置前需要进行安装： sudo apt-get install openssh-server

在本机生成ssh的RSA秘钥： ssh-kgen -t rsa 添加本地ssh公钥到服务器，以实现远程登录: cat ~/.ssh/id_rsa.pub | ssh user@123.45.56.78 “mkdir -p ~/.ssh && cat » ~/.ssh/authorized_keys” 也可以使用，更加快捷： ssh-copy-id user@123.45.56.78

解决ssh需要输入密码和掉线的问题

  • 用了一段时间后，ssh自身的密码可能会过期，需要重新输入，如果输错了那就不能免密登录 Enter passphrase for /Users/username/.ssh/id_rsa

这时可以在客户端运行如下命令将私钥加入到认证客户端中： ssh-add

  • 某些可能网络的原因，ssh会经常断线导致工作不那么流畅，可以尝试修改/etc/ssh/sshd_config，添加如下两个配置：

    ClientAliveInterval 300 ClientAliveCountMax 3

然后重启ssh服务： sudo /etc/init.d/ssh restart

ClientAliveInterval指定间隔多少秒发送一个心跳来使连接保活。 ClientAliveCountMax指定发送多少个心跳后，如果还是没反应就真的断开连接。

客户端也可以设置一个针对服务端的心跳间隔，可以应对服务端没有设置的情况： ServerAliveInterval 300

安装Nginx

ubuntu的软件包源列表中nginx版本是1.4.6，最新的都是1.12了，可以采用官网介绍的办法进行安装。

也可以下载源码自行编译并安装。

从源码安装过程中提示缺少PCRE(用于配置文件正则表达式解析)、zlib、openssl： sudo apt-get install libpcre3 libpcre3-dev zlib1g-dev openssl libssl-dev

安装完成后从外部还是访问不到，发现需要去阿里云的实例管理中添加安全组规则。

Git服务搭建与自动部署

创建新的用户git

sudo adduser git

同样再次配置ssh秘钥登录，创建工程文件目录：

mkdir project.git cd project.git git init –bare

然后在本机中配置远程git仓库,并将工程上传到服务器分支中： git remote add origin git@gitserver:/opt/git/project.git git push –set-upstream origin master
