配置

系统配置

git config --system --edit

当前全局配置

$ git config --global --edit

当前仓库配置

如果需要对某个仓库进行单独配置，那么去掉–global选项，git会采用默认的–local选项，即对当前仓库进行配置。实际工作中，我将真名配置为全局的，以免发生差错。在自己的GitHub仓库中配置为用户名，这都凭个人喜好了，至少配置好全局的就可以开始使用了。

配置用户名与邮箱

一般开始使用时配置用户名和邮箱即可：

$ git config --global user.name "用户名"
$ git config --global user.email "邮箱"

ssh连接

使用ssh连接远程Git仓库，可以参考使用SSH keys进行连接。

https连接

使用HTTPS克隆GitHub仓库，每次与服务器交互都需要输入密码，通过配置让Git缓存密码一定时间：

$ git config --global credential.helper cache
# 设置git使用凭证内存缓存，默认缓存15分钟
$ git config --global credential.helper 'cache --timeout=3600'
# 设置缓存时间为1小时

本地获取远程分支

这个博客的本地仓库原来放在装Ubuntu系统的笔记本中，包含master和post两个分支，post分支用于平时修改预览，写好后再将要发表的内容合并到master分支，再推到远程仓库的master分支，因此远程仓库暂时没有同步post分支。现在用Macbook做开发，希望从远程仓库获取两个分支，于是需要在Ubuntu上将原来post分支也同步到远程。 那么需要设置本地post分支的上游，然后就可以同步post分支内容到服务端，但远程仓库还没有post分支，需要建立该分支，并指定为本地post分支的上游。首先切换到本地post分支，然后指定远程分支origin/post为本地分支的上游:

$ git checkout post
$ git branch -u origin/post

注意指定上游的命令中省略了本地分支名post，是因为已经切换到post分支，如果省略那么就默认为当前分支。

然后推送post分支内容到服务端：

$ git push

OK，现在服务端仓库是最新的了，现在希望在Mac上同步两个分支，由于之前Mac上已经克隆了远程仓库，Mac上的本地仓库仅有master仓库，且不知道服务器端更新后的分支信息。虽然可以使用与上面类似的方式，先在本地建立post分支，然后指定其上游再拉取数据。懒是coder的优点之一。参考Git Pro3.5节，可以直接跟踪远程分支：

$ git checkout -t origin/post

但是出错了：

fatal: Cannot update paths and switch to branch 'post' at the same time.
Did you intend to checkout 'origin/post' which can not be resolved as commit?

google了一个stackoverflow中的答案，此时本地不清楚远程分支的信息。checkout -t不会自动从服务器获取远程分支的信息，需要先在本地更新远程分支的信息：

$ git remote update
