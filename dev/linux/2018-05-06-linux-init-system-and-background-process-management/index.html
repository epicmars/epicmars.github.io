Linux init系统与后台进程管理

什么是init系统

计算机的启动过程从主板上的BIOS（Basic Input / Output System）软件开始，在完成了硬件的初始化和检查后，BIOS启动引导加载程序(bootloader)，bootloader访问存储设备上的主引导记录(Master Boot Record, MBR)，MBR通常是一个硬件驱动，利用这一记录数据定位并启动linux内核。

init系统是内核启动后运行的第一个进程。实时上init系统总是一个系统中进程ID(PID)为1的进程。然后init系统必须启动所有必须的守护进程(daemons)，后台进程和服务，从而使操作系统正常工作。

在完成系统初始化后，init系统继续保持运行，并且管理已经启动的进程，在系统运行过程中可以使得这些进程启动，停止和重启。

常见的init系统

  • System V是一个较老的init系统
    • Debian 6和更早
    • Ubuntu 9.04和更早
    • CentOS 5和更早
  • Upstart
    • Ubuntu 9.10 到 Ubuntu 14.10, 包括Ubuntu 14.04
    • CentOS 6
  • Systemd是最新发布版所采用的init系统
    • Debian 7 和 Debian 8
    • Ubuntu 15.04 和更新
    • CentOS 7

运行级别（runlevel）

  • 0 系统关机
  • 1 单用户，恢复模式
  • 2,3,4：多用户，开启网络，文本模式
  • 5 多用户，开启网络，图像模式
  • 6 系统重启
