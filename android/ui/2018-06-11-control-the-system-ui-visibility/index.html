控制Android系统UI的可见性

系统栏（System bars）分为状态栏（status bar）和导航栏（Navigation bar）。

状态栏 导航栏

使系统栏变暗(dim)

这种方式不会使内容重新调整大小，但系统栏中的图标在视觉上会变弱。只要用户触摸到了系统栏区域，它们就会重新变为完全可见。

改变状态栏

隐藏状态栏

  • 在Android4.0或更低版本上隐藏状态栏

  • 在Android4.1或更高版本上隐藏状态栏

    使用setSystemUiVisibility()在单独的view层级设置UI标志，这些设置整合到window层级。使用这个方法比使用WindowManager可以对系统栏进行更加粒度化的控制。

        View decorView = getWindow().getDecorView();
    // Hide the status bar.
    int uiOptions = View.SYSTEM_UI_FLAG_FULLSCREEN;
    decorView.setSystemUiVisibility(uiOptions);
    // Remember that you should never show the action bar if the
    // status bar is hidden, so hide that too if necessary.
    ActionBar actionBar = getActionBar();
    actionBar.hide();

    需要注意的是：

    • 一旦UI标志被清除（例如从activity导航到其它页面），你的app需要重新设置该标志才能重新隐藏状态栏。参考响应UI可见性的变化，它讨论了在系统UI可见性发生变化时app如何做出相应的响应。
    • 在你设置UI标志并引起改变的地方。如果你在activity的onCreate()方法中隐藏系统栏并且用户按下Home键，那么系统栏会重新出现。当用户重新打开activity时，onCreate()不会被调用，那么系统栏仍然保持可见。如果你希望保持对系统栏的改变，那么你应当在onResume()或onWindowFocusChanged()方法中设置UI标志。
    • 仅仅当调用setSystemUiVisibility()方法的view可见时才有效。
    • 用户在页面间导航离开调用setSystemUiVisibility()的view后会导致设置的标志被清除。

使内容显示在状态栏后面

在Android4.1及更高的版本中，使用系统UI标志SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN可以做到这一点，也可能需要使用SYSTEM_UI_FLAG_LAYOUT_STABLE标志来保持稳定的布局。

使用这种方法需要注意不要让系统栏遮住界面中的关键区域，可以通过在布局文件中添加android:fitsSystemWindows并设置为true。

在默写情况下，你或许需要修改默认的padding。为了直接控制内容如何相对于系统栏进行布局（它占据window中称为"内容嵌入"的一个空间），重写fitSystemWindows(Rect insets)方法，改方法在window的内容嵌入区发生改变时被调用，它允许window据此调整其内容，这样你就可以按你希望的样子操作你的嵌入视图（即应用的布局）了。

改变导航栏

隐藏导航栏

使用SYSTEM_UI_FLAG_HIDE_NAVIGATION可以隐藏导航栏。下面的代码片段同时隐藏导航栏和状态栏。

View decorView = getWindow().getDecorView();
// Hide both the navigation bar and the status bar.
// SYSTEM_UI_FLAG_FULLSCREEN is only available on Android 4.1 and higher, but as
// a general rule, you should design your app to hide the status bar whenever you
// hide the navigation bar.
int uiOptions = View.SYSTEM_UI_FLAG_HIDE_NAVIGATION
              | View.SYSTEM_UI_FLAG_FULLSCREEN;
decorView.setSystemUiVisibility(uiOptions);

这里通过调用view的setSystemUiVisibility()方法的效果和设置状态栏的效果是一致的，有一点不同的是使用这种方法时，当用于触摸屏幕的任意地方会引起导航栏和状态栏重新出现并保持下去。用户交互会导致标志被清除。

使内容显示在导航栏后

在Android 4.1及更高版本中，可以使用SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION标志使应用的内容显示在导航栏后，你可能还需要使用SYSTEM_UI_FLAG_LAYOUT_STABLE标志来帮助app保持一个稳定的布局。

全屏模式

Android提供了三种全屏模式：背靠模式（Lean Back）、沉浸模式（Immersive）、粘性沉浸模式（Immersive Sticky）。三种模式都隐藏了系统栏，区别在于用户如何将系统栏带回到视图。

  • Lean back

    轻触屏幕就可以唤回系统栏。适用于不需要与屏幕频繁交互的场景。

  • Immersive

    从屏幕边缘滑动可以唤回系统栏。适用于需要与屏幕频繁交互的场景。

  • Sticky immersive

    从屏幕边缘滑动可以唤回系统栏，但系统栏是半透明的，并且不会妨碍与App的正常交互，即系统栏区域的触控事件仍然会传递给App。

@Override
public void onWindowFocusChanged(boolean hasFocus) {
    super.onWindowFocusChanged(hasFocus);
    if (hasFocus) {
        hideSystemUI();
    }
}

private void hideSystemUI() {
    // Enables regular immersive mode.
    // For "lean back" mode, remove SYSTEM_UI_FLAG_IMMERSIVE.
    // Or for "sticky immersive," replace it with SYSTEM_UI_FLAG_IMMERSIVE_STICKY
    View decorView = getWindow().getDecorView();
    decorView.setSystemUiVisibility(
            View.SYSTEM_UI_FLAG_IMMERSIVE
            // Set the content to appear under the system bars so that the
            // content doesn't resize when the system bars hide and show.
            | View.SYSTEM_UI_FLAG_LAYOUT_STABLE
            | View.SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION
            | View.SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN
            // Hide the nav bar and status bar
            | View.SYSTEM_UI_FLAG_HIDE_NAVIGATION
            | View.SYSTEM_UI_FLAG_FULLSCREEN);
}

// Shows the system bars by removing all the flags
// except for the ones that make the content appear under the system bars.
private void showSystemUI() {
    View decorView = getWindow().getDecorView();
    decorView.setSystemUiVisibility(
            View.SYSTEM_UI_FLAG_LAYOUT_STABLE
            | View.SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION
            | View.SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN);
}

实战

有时我们需要将UI设置为全屏模式，此时需要将系统的半透明状态栏改为完全透明。这时问题就来了，如果UI是深色背景，那可能没什么问题，因为系统栏图标默认是白色的。如果UI背景是白色的，那么就无法看到状态栏的图标了，这时我们希望能够相应地将状态栏的图标颜色改为深色。
