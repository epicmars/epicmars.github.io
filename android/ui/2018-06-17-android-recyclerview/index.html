NestedScrollView中的RecyclerView

RecyclerView实现了NestedScrollingChild，而NestedScrollView实现了NestedScrollingParent，当RecyclerView嵌套在NestedScrollView中时，其滑动会分发到其NestedScrollingParent，即作为其父布局的NestedScrollView，而NestedScrollView本身也是一个NestedScrollingChild，这一滑动又会向上进行分发。另一方面，NestedScrollView作为NestedScrollingParent又会向下分发滑动事件，这样RecyclerView和NestedScrollView之间的滑动事件分发形成了一个死循环。

ScrollView类的文档中是这样说明的：

    <p>Never add a {@link android.support.v7.widget.RecyclerView} 
    or {@link ListView} to a scroll view. Doing so results in poor 
    user interface performance and a poor user experience.</p>

说明这两者是不兼容的。

实际中遇到的问题是，这样使用会使得RecyclerView的回收策略失效，可能是在NestedScrollView中，RecyclerView总是保留完整的视图项列表，待分析。
