LiteRefresh：一种实现下拉刷新的新方式

项目地址：https://github.com/jastrelax/LiteRefresh

什么是LiteRefersh

LiteRefresh是一个轻量级的嵌套滑动UI库，支持下拉刷新与上拉加载更多。它将布局划分为Header、Content、Footer三部分，将Content作为滑动的轴心，使得Header和Footer可以对Content的滑动作出响应，并且通过一个内置的状态机实现下拉刷新和上拉加载更多。

虽然是UI库但并没有定义新的View或ViewGroup，而是基于Android支持库中已有的CoordinatorLayout进行实现的，因此只要Android支持库的CoordinatorLayout不过时那么这个库也基本不会过时。使用起来也很简单，只需要给CoordinatorLayout的直接子View附加不同的行为也就是Behavior即可，从而达到添加新的嵌套滑动特性的目的。

为什么需要LiteRefresh

开源社区已经有很多优秀的下拉刷新组件了，并且可以做出了许多十分炫酷的效果，那么为什么还需要这个库？正如名称中的Lite，事实上开发这个库的初衷也是为了实现一个简洁的下拉刷新组件，而利用CoordinatorLayout和Behavior可以很容易地实现这一点。但开发过程中发现，CoordinatorLayout远比想象中要强大，正如其名为协调布局所具有的特性，尤其适合需要不同View进行互动的情形。事实上它是所有父布局中对嵌套滑动事件分发的支持是最好的，从而可以通过给其直接子View附加自定义的Behavior来扩展嵌套滑动的行为。此外它的直接子View还可以锚定其它直接子View，而这一点不需要附加额外的Behavior就可以实现，仅需要在布局中定义要锚定的View的id即可，配合这一点可以进一步增强Behavior所实现的效果。

因此最初的初衷从实现一个简洁的下拉刷新组件变成了一个更为通用的嵌套滑动组件，而下拉刷新只是其必须要支持的重要特性之一，因为绝大多数APP都是有这种从网络刷新或加载更多数据的需求的。

调研

开发过程中，首先对许多主流APP的主要页面进行调研：

  • 支付宝首页：下拉刷新，刷新的Loading动画在中间，上滑使头部视图折叠为一个工具栏
  • 微博首页列表：较为经典的下拉刷新与上拉加载更多
  • 微信发现、设置等页面：列表在上下滑动可以滑出原始边界
  • 微信朋友圈页面：包含刷新指示的头部部分可见，并且可以跟随嵌套滑动内容，就像它们是一个整体一样
  • 猫眼电影主页：有一个锚定列表的工具栏，向上滑动到顶部时保持位置不变，向下滑动时跟随列表滑动
  • 百度贴吧个人主页：下拉时头部图片背景可以进行缩放
  • 等等

经过总结，发现包括上述页面在内的绝大多数页面都可以划分为以下4个部分：

  • Content（内容）

    有一个总是可以滑动的列表，它主要用于信息的展示

  • Header（头部）

    头部的功能有很多种，如：

    • 默认隐藏，主要在下拉刷新中作为刷新的指示器，下拉时可见
    • 初始完全可见或部分可见，主要用于展示与Content不一样的内容，Content滑动时可以跟随也可以不跟随，还可以选择在Content滑动时实现一些特效，如

  • Footer（脚部）

    Footer的特征与Header是类似的，但有一点不一样的是它很少作为内容的展示来使用：

    • 默认隐藏，主要在上拉加载更多中作为加载指示器，当Content滑动到尾部时上拉可见

  • 网络刷新或加载指示器

    一般情况下，指示器一般是由Header和Footer实现的，但遇到的很多页面中并不是，这时其位置是不确定的，有可能跟随在Content的上方，也有可能位于顶部被Header所遮挡，还存在各种其它可能的情况等等。

    因此在LiteRefresh中，对指示器的支持也是与其它下拉刷新库不同的地方，即Header与Footer的主要功能并不一定是指示器。实际上LiteRefresh并没有专门定义指示器相关组件，指示器的实现是相当灵活的，它可以是包括Header与Footer在内的任何View，可以出现在界面的任何地方，只要监听刷新或加载状态就可以了。

Behavior

最终实现的特性可以很容易地实现上述各种类型的UI效果，这是使用三种不同的Behavior来实现的，其名称与特性如下：

  • RefreshContentBehavior

    • 可以让Content滑出初始布局边界，并且可以限定滑出的范围
    • 可以让Content部分可见或者反过来说让其部分不可见
    • 可以添加滑动监听器监听Content滑动的偏移量变化
    • 可以添加下拉刷新状态监听器来观察刷新状态的改变
    • 可以添加上拉加载更多的状态监听器来观察加载状态的改变
    • 它是唯一必须的Behavior，并且可以单独使用

  • RefreshHeaderBehavior

    • 可以让Header部分可见、完全可见或完全隐藏
    • 可以让Header选择跟随或者不跟随Content一起滑动，有4种模式供选择
    • 可以添加滑动监听器来观察Header滑动偏移量的变化
    • 可以添加下拉刷新状态监听器来观察刷新状态的变化
    • 它是可选的

  • RefreshFooterBehavior

    • 可以让Footer部分可见、完全可见或完全隐藏
    • 可以让Footer选择跟随或者不跟随Content一起滑动，有4中模式供选择
    • 可以添加滑动监听器来观察Footer滑动偏移量的变化
    • 可以添加上拉加载状态监听器来观察加载状态的变化
    • 它也是可选的

Header与Footer的滑动模式

上面说到Header与Footer可以选择跟随或者不跟随Content一起滑动，一共定义了4种模式供选择：

  • Follow：这是默认的模式即总是跟随Content一起滑动
  • Follow up：仅当Content向上滑动时才跟随，否则保持静止
  • Follow down：仅当Content向下滑动时才跟随，否则保持静止
  • Still：不跟随Content滑动，总是保持静止

使用

使用起来也是很简单，只需将上述三种Behavior附加到CoordinatorLayout的直接子View即可，注意是直接子View，并且作为Content的View必须是支持嵌套滑动事件分发的，支持最好的是RecyclerView和NestedScrollingView，它们都实现了NestedScrollingChild接口。

在布局中使用

单独使用Content Behavior

在CoordinatorLayout下，将作为Content的View的app:layout_behavior属性设置为@string/lr_refresh_content_behavior即可。

    <android.support.design.widget.CoordinatorLayout
        xmlns:android="http://schemas.android.com/apk/res/android"
        xmlns:app="http://schemas.android.com/apk/res-auto"
        android:id="@+id/coordinator_layout"
        android:layout_width="match_parent"
        android:layout_height="match_parent">

        <android.support.v7.widget.RecyclerView
            android:id="@+id/recycler_view"
            android:layout_width="match_parent"
            android:layout_height="match_parent"
            app:layout_behavior="@string/lr_refresh_content_behavior">
        </android.support.v7.widget.RecyclerView>
    </android.support.design.widget.CoordinatorLayout>

同时使用Content、Header、Footer Behavior

以下是示例APP中的展示Follow模式的一个例子，注意到其中还有一个SampleAnchoredView，它利用到了CoordinatorLayout中锚定其它View的特性。

    <android.support.design.widget.CoordinatorLayout
        xmlns:android="http://schemas.android.com/apk/res/android"
        xmlns:app="http://schemas.android.com/apk/res-auto"
        android:id="@+id/coordinator_layout"
        android:layout_width="match_parent"
        android:layout_height="match_parent">

        <com.androidpi.literefresh.sample.ui.widget.SampleHeaderView
            android:id="@+id/view_header"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            app:layout_behavior="@string/lr_refresh_header_behavior"
            app:lr_maxOffsetRatio="100%p"
            app:lr_mode="follow"
            app:lr_visibleHeightRatio="50%" />

        <com.androidpi.literefresh.sample.ui.widget.SampleFooterView
            android:id="@+id/view_footer"
            android:layout_width="match_parent"
            android:layout_height="150dp"
            app:layout_behavior="@string/lr_refresh_footer_behavior"
            app:lr_maxOffsetRatio="100%p" />

        <com.androidpi.literefresh.sample.ui.widget.SampleContentView
            android:id="@+id/view_content"
            android:layout_width="match_parent"
            android:layout_height="match_parent"
            app:lr_minOffset="64dp"
            app:layout_behavior="@string/lr_refresh_content_behavior"/>

        <com.androidpi.literefresh.sample.ui.widget.SampleAnchoredView
            android:id="@+id/view_anchored"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            app:layout_anchor="@id/view_content"
            app:layout_anchorGravity="top|center_horizontal" />
    </android.support.design.widget.CoordinatorLayout>

在Java代码中使用

在CoordinatorLayout.LayoutParams中设置Behavior

    CoordinatorLayout.LayoutParams params = ((CoordinatorLayout.LayoutParams) getLayoutParams());
    RefreshHeaderBehavior headerBehavior = new RefreshHeaderBehavior(context);
    params.setBehavior(headerBehavior);

自定义View实现CoordinatorLayout.AttachedBehavior接口

public class RefreshHeaderLayout extends FrameLayout implements CoordinatorLayout.AttachedBehavior{

    protected RefreshHeaderBehavior behavior;
    
    // ...

    public RefreshHeaderLayout(Context context, AttributeSet attrs, int defStyle) {
        super(context, attrs, defStyle);
        behavior = new RefreshHeaderBehavior(context, attrs);
    }

    @NonNull
    @Override
    public CoordinatorLayout.Behavior getBehavior() {
        return behavior;
    }
}

配置

  • Content

    名称 描述 布局属性
    最大偏移量 Content的顶部相对于父布局顶部的最大偏移量 lr_maxOffset , lr_maxOffsetRatio
    最小偏移量 Content的顶部相对于父布局顶部的最小偏移量 lr_minOffset , lr_minOffsetRatio
    头部可见高度 如果Content单独使用，它表示Content的逻辑头部可见高度 lr_headerVisibleHeight
    头部触发偏移量 如果Content单独使用，它表示Content的逻辑头部触发偏移量 lr_headerTriggerOffset
    脚部可见高度 如果Content单独使用，它表示Content的逻辑脚部可见高度 lr_footerVisibleHeight
    脚部触发偏移量 如果Content单独使用，它表示Content的逻辑脚部触发偏移量 lr_footerTriggerOffset

  • Header

    名称 描述 布局属性
    最大偏移量 Header的底部相对于父布局顶部的最大偏移量，它是与Content的最大偏移量相关的 lr_maxOffset , lr_maxOffsetRatio
    可见高度 Header可以被看见的高度 lr_visibleHeight, lr_visibleHeightRatio
    触发偏移量 Header从初始位置向下滑动可以触发刷新的距离 lr_triggerOffset

  • Footer

    名称 描述 布局属性
    最大偏移量 Footer的顶部相对于父布局底部的最大偏移量 lr_maxOffset , lr_maxOffsetRatio
    可见高度 Footer可以被看见的高度 lr_visibleHeight, lr_visibleHeightRatio
    触发偏移量 Footer从初始位置向上滑动可以触发刷新的距离 lr_triggerOffset
