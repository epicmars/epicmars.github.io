Android应用兼容性问题

Android系统Native库

Android系统版本分布的碎片化导致所使用的本地库版本的碎片化，有可能会使用相当过时的本地库。

  • SQLite

    SQLite目前官方最新的版本已经是3.8.0，逐步增加对一些SQL新特性的支持，例如with子句，但在使用AS自带模拟器运行Android4.4进行测试时发现，在Android5.1的真机上运行没问题的一个带with子句的查询抛出了一个syntax error异常，发现是系统SQLite Native库版本较低，还不支持较新的特性。

因此，使用这些Native库时还是应该作好调查，Android平台版本本身碎片化严重，在使用新特性方面最好不要过于激进。

App引入的第三方库

应用中引入的第三方库可能存在未发现或未修复的bug，也有可能bug在最新的版本中已经修复，但应用中仍然使用了过时的版本。

例如RxJava，目前常用的是RxJava2，Bug修复和版本发布都挺频繁的，RxJava2严格按照响应式编程的规范进行开发，对于一个Subscriber：

public interface Subscriber<T> {
    public void onSubscribe(Subscription s);
    public void onNext(T t);
    public void onError(Throwable t);
    public void onComplete();
}

如果它在onSubscribe, onNext, onError或onComplete中传入的参数为null那么就必须抛出一个java.lang.NullPointerException异常。而实际中，RxJava库的实现中可能存在的Bug导致它没有抛出异常，这时就会导致意料之外的情况的发生。
