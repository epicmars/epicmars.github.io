Jenkins配置及其与Gerrit的集成

Gerrit配置

  1. 建立jenkins用户

    首先需要创建一个jenkins用户用于代表Jenkins服务器与Gerrit进行交互，目前Gerrit服务器使用的公司的域账号进行用户认证，但新建的jenkins用户并不需要使用域账号就可以建立。

    参考gerrit create-account和gerrit set-account:

    ssh -p 29418 review.example.com gerrit create-account --http-password jenkins jenkins

# 首先将Jenkins服务器的ssh公钥内容拷贝到本地的jenkins.pub文件
cat jenkins.pub | ssh -p 29418 review.example.com gerrit set-account --add-ssh-key - jenkins

  2. 设置Jenkins访问权限

    根据Gerrit Trigger，将jenkins用户加入Non-Interactive Users组，在需要进行构建的项目访问权限中添加权限：

    • Reference: refs/*

      Read: ALLOW for Non-Interactive Users

    • Reference: refs/heads/*

      Label Code-Review: -1, +1 for Non-Interactive Users

      Label Verified: -1, +1 for Non-Interactive Users

    在配置过程中发现没有Label Verified选项，这需要编辑All-Project下的project.config文件：

    [label "Verified"]
    function = MaxWithBlock
    value = -1 Fails
    value = 0 No score
    value = +1 Verified
    copyAllScoresIfNoCodeChange = true

    在Gerrit2.7及以上版本，还需要保证“Stream Events”可用，否则该插件会反复连接Gerrit导致gerrit出现OOM错误。创建新的组“Event Streaming Users”，将jenkins加入其中，然后在All-Projects的访问控制中编辑：

    • Global Capabilities

      Stream Events: ALLOW for Event Streaming Users

添加构建脚本

ci.gragle

    def getKeystorePropertiesFile() {
        def keystorePropertiesFile = rootProject.file("keystore.properties")
        if (!keystorePropertiesFile.exists()) {
            def EXAMPLE_APP_KEYSTORE_DIR = System.getenv("EXAMPLE_APP_KEYSTORE_DIR")
            keystorePropertiesFile = new File(EXAMPLE_APP_KEYSTORE_DIR, "keystore.properties")
        }
        return keystorePropertiesFile
    }

    def keystoreProperties = new Properties()
    keystoreProperties.load(new FileInputStream(getKeystorePropertiesFile()))

    ext.keystoreProperties = keystoreProperties
