Java多线程：synchronized

Java语言提供了多种线程间通信的方式，最基本的是同步，它是使用监视器（moniter）实现的。Java中的每个对象都与一个监视器相关联，线程可以对该监视器进行加锁和解锁。在一个时刻仅一个线程可以持有一个监视器上的一个锁，其它任何尝试对该监视器加锁的线程都会被阻塞，知道它能够获取到该监视器上的一个锁。一个线程t可以多次对一个特定的监视器加锁，每个解锁操作会解除一个加锁操作说施加的效果。

synchronized语句与方法

synchronized语句计算对一个对象的引用，然后尝试在该对象的监视器上进行加锁操作，如果不能成功获取到锁那么就不会进行下一步的处理。在加锁动作执行后，synchronized语句就会执行，无论语句正文完成，不论是异常还是正常完成，都会在相同的监视器上自动执行解锁的操作。

synchronized方法在被调用时自动进行加锁操作，如果不能成功获取到锁就不会执行方法体。如果方法是实例方法，它会对被调用了该方法的对象（也就是在执行方法体的过程中作为this的对象）所关联的监视器进行加锁。如果方法是static的，那么它会对方法说定义的类的Class对象关联的监视器进行加锁。一旦方法执行完成，无论正常还是异常结束，都会在相同的监视器上自动执行解锁的操作。

synchronized语句

一个synchronized语句为正在执行的线程获取一个互斥锁，执行一个块，然后释放说。在执行线程持有锁期间，其它线程都不能获取该锁。

  SynchronizedStatement:
      synchronized ( Expression ) Block

Expression的类型必须是一个引用类型，否则会发生编译时错误

一个synchronized语句会首先对Expression进行计算，然后：

  • 如果对Expression的计算异常结束，那么synchronized语句会以相同的原因结束
  • 否则，如果Expression的值是null，那么会抛出NullPointerException
  • 否则，如果non-null的Expression的值是V，那么执行线程会对V关联的监视器加锁。然后Block得到执行，然后需要作出一个选择：
    • 如果Block的执行正常结束，那么监视器解锁并且synchronized语句正常结束
    • 如果Block的执行异常结束，那么监视器解锁并且synchronized语句以同样的原因异常结束 synchronized语句的获取的锁和synchronized方法隐式获取的锁一样。单个线程可以可以多次获取一个锁。

synchronized方法

一个synchronized方法在执行前获取一个监视器。

对于一个类（static）方法，会使用方法所在类的Class对象所关联的监视器。

对于一个实例方法，会使用this（该方法被调用的对象）所关联的监视器。
