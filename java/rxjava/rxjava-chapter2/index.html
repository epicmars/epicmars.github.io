Observable剖析

rx.Observable

使用订阅（subscription）和订阅者（Subscriber）来控制监听器

单个Observable可以拥有多个订阅者。

创建Observable

无限流

在create中不应当显式地创建线程，Rx设计指南中假设observer实例以串行的方式调用，这就使得订阅者从来不会并行地接收到消息。

热的与冷的Observable

热的（Hot）Observable不同于与冷的Observable，在你拿到这样的一个Observable，无论它有多少个订阅者，它可能已经在发送事件了。
